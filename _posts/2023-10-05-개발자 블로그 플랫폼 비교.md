> 요약 – 개발자와 CS 전공자들에게 블로그는 지식 공유의 중요한 수단이다. 하지만 현재는 수많은 블로그 플랫폼이 존재하여, 사용자들이 자신에게 적합한 플랫폼을 선택하는 데 어려움을 겪고 있다. 따라서 **GitHub Pages, WordPress, Notion, 네이버 블로그, 티스토리, velog**를 플랫폼 기반 관점, 사용자 기반 관점에서 비교하여 각각의 특성을 알아보고자 한다. **플랫폼 기반 관점**에서는 플랫폼이 가지는 고유한 특성(운영사(자), 제작 국가, 지원 언어, 용도, 회원 가입의 필요성, 마크다운 외의 교육(언어) 필요성)을 바탕으로 비교한다. **사용자 기반 관점**에서는 사용자가 고려할 만한 특성(마크다운 지원, 디자인 자유성, 디자인 편의성, 광고 삽입 가능성, 유료 서비스 금액, 플러그인 지원 여부)을 바탕으로 비교한다. 비교 결과를 통해, 사용자들은 자신의 목적과 필요에 적합한 플랫폼을 선택할 수 있다.

## 서론
개발자들과 CS 전공자들에게 있어 블로그는 정보와 지식을 공유하는 중요한 창구이다. 이러한 중요성에 따라 오늘날에는 수많은 블로그 플랫폼이 존재한다. 따라서 새로운 개발자들이나 블로그를 시작하려는 사람들에게는 블로그 플랫폼의 선택이 힘들 수 있다.  이 글에서는 대표적인 외국 블로그 플랫폼인 **GitHub Pages, WordPress, Notion**과 국내 블로그 플랫폼인 **네이버 블로그, 티스토리, velog**를 비교하고자 한다. 먼저 **플랫폼 기반 관점(platform-based view)**에서 비교하여, 플랫폼의 특징에 대해 각각 살펴본다. 다음으로 **사용자 기반 관점(user-based view)**에서 비교하여 어떤 블로그가 사용자의 관점에서 더 유용한지 살펴본다.

## 본론
이 글에서는 비교할 블로그 플랫폼으로 GitHub Pages, WordPress, Notion, 네이버 블로그, 티스토리, velog를 택한다. 그리고 이들을 플랫폼 기반 관점, 사용자 기반 관점에서 비교해 보며 각 플랫폼의 특징에 대해 알아보고자 한다.

### 1. 플랫폼 기반 관점(platform-based view) 비교
먼저 플랫폼 기반 관점(platform-based view)에서 비교한다.  플랫폼 기반 관점의 비교는 플랫폼이 가지는 고유한 특성을 바탕으로 한 비교를 말한다. 플랫폼의 운영사(자), 제작 국가, 지원 언어, 용도, 회원 가입의 필요성, 마크다운 외의 교육(언어) 필요성을 기준으로 비교를 할 것이다. 이러한 특성을 비교함으로써, 각 플랫폼 자체가 가지는 특징에 대해 알 수 있다.

![플랫폼 기반 관점에서의 비교]({{site.baseurl}}/images/20231005/02.png)
<br>
###### *Figure 2. 플랫폼 기반 관점에서의 비교*

#### 1) 운영사(자)
먼저 운영사(자) 항목에서 보면, GitHub Pages, 네이버 블로그, 티스토리는 이미 자사의 서비스 플랫폼이 존재하고, 이에 대한 확장 서비스로 블로그 플랫폼을 운영함을 알 수 있다. 따라서 만약 사용자가 GitHub, 네이버, 카카오를 이용하고 있다면 자사의 블로그 플랫폼을 이용하는 것이 적합하다.

#### 2 & 3) 제작 국가 & 지원 언어
국외 플랫폼에서는 당연히 영어를   지원한다. 그리고 한국어와 같은 다른 국가의 언어들도 지원한다. 그러나 한국에서 제작된 블로그 플랫폼의 경우에는 한국어만 지원된다. 따라서 한국어 외의 다른 언어가 더 편하거나 외국인과 교류할 예정이라면 GitHub Pages, WordPress, Notion을, 그렇지 않다면 국내 블로그 플랫폼을 이용하는 것이 적합하다.

#### 4) 용도
대부분의 블로그 플랫폼은 블로그를 목적으로 운용되고 있으며 Notion의 경우에는 필기 및 위키, 협업의 목적으로 사용된다. 이 글에서는 블로그를 기준으로 살피고 있으므로, 모든 플랫폼은 용도에 있어서 적절하다.

#### 5) 회원가입의 필요성
모든 플랫폼은 서비스를 이용하기 위해서 회원가입이 필요하다. 따라서 사용자들은 해당 서비스 이용 시 회원가입을 필수로 해야 한다. 물론 회원가입을 하지 않아도 조회는 할 수 있지만, 이 글은 블로그의 운영을 목적으로 하므로, 모든 플랫폼에서 회원가입이 필수라고 볼 수 있다.

#### 6) 마크다운 외의 교육 필요성
대부분의 블로그 플랫폼은 글의 작성에 있어 마크다운 형식으로 글을 작성한다. 하지만 GitHub Pages의 경우에는 Git에 대한 지식이 추가로 있어야 한다. WordPress는 추가로 PHP에 대한 지식이 있어야 한다. 이에 반해, 다른 블로그 플랫폼은 마크다운 형식과 각 서비스의 사용법 정도만 알고 있으면 글을 작성하는 데 큰 무리가 없다. 따라서 사용자들은 본인의 지식 및 경험 수준에 근거하여 블로그 플랫폼을 선택하면 된다.

***

### 2. 사용자 기반 관점(user-based view) 비교
다음으로 사용자 기반 관점(user-based view)에서 비교한다. 사용자 기반 관점의 비교는 사용자가 플랫폼을 이용하는 데 있어 고려할 수 있는 특성을 바탕으로 한 비교를 말한다. 마크다운 지원, 디자인 자유성, 디자인 편의성, 광고 삽입 가능성, 유료 서비스(유료 요금제 중 가장 저렴한 요금제 기준) 금액, 플러그인 지원 여부 기준으로 비교할 것이다.
<br>
여기서 디자인 자유성이란 사용자가 블로그 플랫폼을 사용할 때 본인의 블로그 테마나 글의 테마를 자유롭게 꾸밀 수 있는지를 말한다. 디자인 편의도란 사용자가 얼마나 편리하게 디자인을 할 수 있는지를 말한다. 

![사용자 기반 관점에서의 비교]({{site.baseurl}}/images/20231005/03.png)
<br>
###### *Figure 3. 사용자 기반 관점에서의 비교*

#### 1) 마크다운 지원
사용자가 문서 제작을 할 때 더욱 쉽게 할 수 있게 해주기 위해서 마크다운은 필수적이다. 또한 개발자나 CS 전공자들에게는 ‘코드 입력’과 같은 기능도 있으므로 더욱 필수적이다. 대부분의 플랫폼에서는 마크다운을 지원하고 있지만, 네이버 블로그는 지원하지 않는다.

#### 2) 디자인 자유성
GitHub Pages, WordPress, 네이버 블로그, 티스토리는 블로그 및 글을 제작하는 데 사용자의 취향에 맞게 디자인을 수정할 수 있다. 하지만 Notion, velog는 플랫폼 자체의 디자인을 수정할 수 없다. 따라서 사용자가 본인의 디자인을 구축하고자 한다면 GitHub Pages, WordPress, 네이버 블로그, 티스토리를 이용하는 것이 적절하고, 그렇지 않다면 Notion, velog의 이용도 고려해 볼 수 있다.

#### 3) 디자인 편의도
디자인 편의도는 디자인 자유성이 있는 플랫폼에서 높은 편의성을 보이고, 디자인 자유성이 없는 플랫폼에서는 낮은 편의성을 보였다. 디자인 자유성이 있는 플랫폼에서는 블로그 테마, 글의 템플릿과 같은 기능을 제공하여 사용자가 디자인함에 있어 겪을 수 있는 막연함을 해소한다.

#### 4) 광고 삽입 가능성
사용자들은 블로그 운영과 동시에 수익 창출을 원할 수 있다. Notion과 velog는 광고 삽입을 하지 못한다. 물론 사용자가 광고주와 파트너십을 맺어 광고를 게재할 수는 있지만, 플랫폼 자체적으로 플러그인이나 특정 기능을 제공하고 있지 않다. 따라서 사용자가 광고를 통해 수익을 창출하길 원한다면, 위의 두 플랫폼을 제외한 나머지 플랫폼이 적절하다.


#### 5) 유료 서비스의 금액
유료 서비스는 금액별로 다양한데, 비교를 위해 유료 서비스 중 가장 저렴한 서비스를 기준으로 선정했다. 위의 플랫폼들은 무료 요금제도 지원하고 있지만, 사용자들이 더 확장된 서비스를 이용하기 위해 유료 서비스에 가입할 수도 있다. 국내 플랫폼인 네이버 블로그, 티스토리, velog는 유료 서비스가 없다. 그러나 외국 플랫폼인 GitHub Pages, WordPress는 한 달에 4달러, Notion은 한 달에 8달러를 부과한다. 이때, GitHub Pages, WordPress의 요금제는 개인을 대상으로 하고, Notion은 소규모 단체를 대상으로 하기 때문에 금액의 차이는 당연하다.


#### 6) 플러그인 지원 여부
사용자들은 여러 플러그인을 통해 쉽고 확장된 블로그 서비스를 이용할 수 있다. 네이버 블로그, velog는 이러한 플러그인 지원이 되지 않으므로, 사용자들은 플랫폼 자체의 기능만 사용할 수밖에 없다. 만약 사용자들이 플러그인이 필요 없고 간단한 작업만 수행한다면 위의 두 플랫폼을 이용해도 무방할 것이다. 하지만 보다 많은 기능을 사용하고자 한다면 나머지 플랫폼을 이용하는 것이 적절할 것이다.

## 결론
지금까지 다양한 블로그 플랫폼을 플랫폼 기반 관점(platform-based view)과 사용자 기반 관점(user-based view)에서 비교했다.
<br>
플랫폼 기반 관점에서는 플랫폼이 가지는 고유한 특성을 바탕으로 비교를 진행했다. 만약 사용자가 GitHub, 네이버, 카카오와 같은 서비스를 이용한다면 GitHub Pages, 네이버 블로그, 티스토리 사용이 적절할 것이다. 또한 사용자가 한국어가 아닌 외국어에 더 친숙하거나 외국 사용자와 소통을 자주 한다면, 외국 플랫폼인 GitHub Pages, WordPress, Notion이 더 적합하다. 또한, 사용자가 Git에 대한 지식이 있다면 GitHub Pages, PHP에 대한 지식이 있다면 WordPress를 사용해도 무방하다. 하지만 그러한 지식이 없다면 다른 플랫폼을 이용하는 것이 적합하다.
<br>
사용자 기반 관점에서는 사용자가 고려할 만한 특성을 바탕으로 비교를 진행했다. 디자인을 자유롭게 하고 싶다면 GitHub Pages, WordPress, 네이버 블로그, 티스토리가 적합하다. 블로그를 운영하면서 광고를 통해 수익을 창출하고 싶다면, GitHub Pages, WordPress, 네이버 블로그, 티스토리가 적합하다. 또한, 모든 서비스를 무료로 사용하고자 한다면 국내 플랫폼을 사용하는 것이 적합하다. 다양한 플러그인을 사용하고자 한다면 GitHub Pages, WordPress, Notion, 티스토리를 사용하는 것이 적합하다.
<br>
각 블로그 플랫폼은 고유한 장점과 단점을 가진다. 따라서 **사용자들은 자신들의 목적과 필요에 따라 가장 적합한 플랫폼을 선택하는 것이 중요**하다. 이때, 사용자들은 위와 같은 비교 결과를 참고한다면, 더욱 효율적으로 블로그 플랫폼의 선택 및 운영을 할 수 있을 것이다.
